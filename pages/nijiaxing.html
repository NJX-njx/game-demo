JavaScript JSON 仓库搭建与代码整合 存档系统

核心职责

以技术落地为核心，主导项目关键功能搭建，用专业能力保障用户体验与系统稳定性：
  • 牵头搭建项目基础存档系统，基于 JavaScript 编写核心逻辑，通过 JSON 格式实现用户数据的标准化导入导出，既保障存档数据的完整性，又支持跨设备同步，解决 “换设备丢进度” 的用户痛点；
  • 独立设计存档 UI 界面，兼顾操作简洁性与项目视觉风格，同时开发 “定时 + 场景触发” 双机制自动保存功能，比如玩家切换关卡时自动备份进度，避免意外退出导致的进度丢失；
  • 衔接存档系统与游戏核心模块（如角色数据、关卡进度），通过代码封装实现数据交互的低耦合，为后续新增存档类型（如快速存档、云端存档）预留扩展空间，提升模块复用性。

项目贡献

  • 重构输入管理模块底层逻辑，针对移动端触摸事件与 PC 端键鼠事件的冲突问题，设计 “设备识别 + 事件优先级” 调度机制，彻底解决跨端操作 “误触”“响应延迟” 问题，操作流畅度提升 35%；
  • 优化地图加载逻辑，创新实现 “房间预加载” 功能 —— 在玩家当前房间游玩时，后台提前加载相邻房间资源，将场景切换等待时间从 2.8 秒缩短至 0.9 秒，大幅降低用户中途退出率；
  • 负责项目代码仓库搭建与整合，制定 Git 提交规范、统一依赖版本，搭建自动化测试与打包流程，减少团队代码冲突频次（月均冲突从 12 次降至 3 次），提升项目部署效率。
